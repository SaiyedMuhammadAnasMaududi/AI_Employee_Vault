```bash

```

You should see these processes:

| Name | Status | Job |
|---|---|---|
| `ralph` | online | Reads tasks, calls Qwen, sends replies |
| `watchdog` | online | Monitors all processes, auto-restarts crashed ones |

> `errored` = crashed — check logs immediately.

### Quick health check (one command)

```bash
pm2 jlist | python3 -c "
import sys, json
procs = json.load(sys.stdin)
for p in procs:
    name   = p['name']
    status = p['pm2_env']['status']

"
```

### Token expiry

The Meta access token expires approximately **every 60 days**.
When it expires you'll see `HTTP Error 401` in the logs.

To refresh:
1. Go to **developers.facebook.com/tools/explorer/**
2. Select **"Ai app"** → Generate Access Token
3. Check: `pages_show_list`, `pages_read_engagement`, `pages_manage_posts`, `instagram_basic`, `instagram_content_publish`
4. Run: `GET /1036269206231768?fields=access_token`
5. Copy the `access_token` from the result
6. Update `.env`: `META_ACCESS_TOKEN=<new token>`
7. Restart: `pm2 restart linkedin` (or nothing — no restart needed for meta_social_mcp)

## 9. Ralph — Task Processor & HITL Flow

> Ralph is the brain — it reads tasks and decides what to do.

### Start

```bash
pm2 start ralph
```

### Check Ralph is processing

```bash
pm2 logs ralph --lines 30
```

Look for:
```
ralph_wrapper: polling for tasks...
ralph_wrapper: no pending tasks.
```

### How the HITL (Human-in-the-Loop) flow works

```
Task created in /Needs_Action/
  → Ralph reads it
  → If < $50 / known contact → executes automatically
  → If > $50 / new contact / social post → creates file in /Pending_Approval/
       → YOU review and move to /Approved/ or /Rejected/
  → If /Approved/ → Ralph executes the action
  → Moves file to /Done/
```

### Approve a pending action

```bash
# List pending approvals
ls Pending_Approval/

# View it
cat Pending_Approval/<filename>.md

# Approve
mv Pending_Approval/<filename>.md Approved/

# Reject
mv Pending_Approval/<filename>.md Rejected/
```

### View all completed tasks

```bash
ls Done/ | tail -20
```

### Full audit log

```bash
cat Logs/agent_log.md | tail -50
```

## 11. Troubleshooting Quick Reference

| Problem | Fix |
|---|---|
| `pm2 list` shows nothing | Run `pm2 resurrect` |
| Process shows `errored` | `pm2 logs <name> --err --lines 30` to see why |
| Process keeps restarting | `pm2 stop <name>` → fix the error → `pm2 start <name>` |
| Gmail not detecting emails | Check `GMAIL_TOKEN_PATH` in `.env` — token may need refresh |
| LinkedIn session expired | `pm2 stop linkedin` → `python3 auth_setup.py --linkedin` → `pm2 start linkedin` |
| WhatsApp session expired | `pm2 stop whatsapp` → `python3 auth_setup.py --whatsapp` → `pm2 start whatsapp` |
| Meta 401 Unauthorized | Token expired — regenerate via Graph API Explorer (see Section 6) |
| Odoo connection refused | `docker ps` — start Docker if not running |
| Odoo auth failed | Check `ODOO_USER=tafs4920@gmail.com` and `ODOO_API_KEY` in `.env` |
| Ralph not processing tasks | `pm2 logs ralph --err` — check for Qwen CLI issues |
| Qwen not found | `which qwen` — ensure it's on PATH |
| Instagram post failing | Image URL required — default is auto-used if none provided |
| Watchdog restarting stopped process | Only `errored` processes are restarted — `stopped` is respected |

### PM2 cheat sheet

```bash
pm2 list              # see all processes and status
pm2 start <name>      # start a stopped process
pm2 stop <name>       # stop a process (intentional — won't be restarted by watchdog)
pm2 restart <name>    # restart a running process
pm2 restart all       # restart everything
pm2 stop all          # stop everything
pm2 resurrect         # restore last saved state (use after reboot)
pm2 save              # save current state for next resurrect
pm2 logs <name>       # live logs for one process
pm2 logs <name> --lines 50   # last 50 lines
pm2 logs <name> --err        # errors only
```

### Environment variables (`.env`) quick reference

| Variable | What it controls |
|---|---|
| `ANTHROPIC_API_KEY` | Claude API (not used — Qwen is used instead) |
| `GMAIL_APP_PASSWORD` | Gmail SMTP password for sending emails |
| `GMAIL_ADDRESS` | Gmail account being monitored |
| `GMAIL_POLL_INTERVAL` | How often Gmail is checked (default: 120s) |
| `LINKEDIN_AUTO_POST_HOUR` | Hour to post daily (default: 9 = 9 AM) |
| `LINKEDIN_CROSS_POST_FB` | Also post to Facebook + Instagram after LinkedIn (default: true) |
| `META_ACCESS_TOKEN` | Facebook/Instagram Page Access Token |
| `META_PAGE_ID` | Facebook Page ID (My Services: 1036269206231768) |
| `META_IG_ID` | Instagram Business Account ID |
| `META_DEFAULT_IG_IMAGE` | Default image used when no image URL provided |
| `ODOO_URL` | Odoo server URL (default: http://localhost:8069) |
| `ODOO_API_KEY` | Odoo API key |
| `ODOO_HITL_LIMIT` | Auto-proceed threshold in $ (default: 50) |
| `DRY_RUN` | Set to `true` to disable real email sends |
